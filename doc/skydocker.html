docker of python-email-smtp on ubuntu

Demand

üòúPython3.6+ ,ubuntu16.04+ or windows10 1803+

Install

pull from docker hub
run docker
start service

test your mail at port 25 cd /home && ./mail.sh
test your mail at port 465 as ssl cd /home && ./mail-ssl.sh
run shceduled_email at port 25 cd /home && ./mail-back.sh
run scheduled_email at port 465 as ssl cd /home && ./mail-back-ssl.sh

üòÑonce mail.py runs successfully,you will see your python email is running

modify parameter

2 json file are provided to customize

for scheduled_mail,you can change time as you wish,the mail.service will send email every day at this time

Notes

  • mail-back.sh ,mail-back-ssl.shuse nohup to prevent system from killing your python.service
    üò≠so you can not see your python email is runningif using it

  • to use schedule you must install schedule

  • it use smtp.163.com as default smtp,if you want to use other smtp service,you can change it in mail.json,mail-sche.json
  • a json file will soon be add,then you can modify the parameters by yourself
  • some cache will still in your disk after your pull this image,so run apt clean && apt autoremove

Attention

to use smtp service you must active your 25 port on your server
to use smtp through ssl you must active your 465 port on your server
